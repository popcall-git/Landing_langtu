# Layout Patterns

## Default Landing Structure

1. Eyebrow / category label
2. Hero headline
3. Supporting paragraph
4. Primary CTA + secondary CTA
5. Featured content cards
6. Community / context explanation
7. Social proof / comments
8. Final CTA

## Content Mapping Rules

### If source is a category page
- Use the top 2-4 items as featured cards.
- Preserve original titles when they are hook-heavy.
- Add click-through links to the original detail pages.

### If source includes videos
- Use extracted keyframes as card or hero media.
- Prefer local assets instead of fragile blob URLs.

### If source is an article or promo page
- Convert the strongest claims into:
  - headline
  - subheadline
  - benefit bullets
  - CTA copy

## Dark-Impact Pattern

Use when the page needs immediate attention and high contrast.

Characteristics:
- black / charcoal background
- warm red, orange, or magenta accents
- oversized headlines
- large media blocks
- glass or translucent panels used sparingly
- strong CTA buttons

## Social Proof Pattern

If real comments are unavailable, create clearly synthetic but plausible comment-style copy blocks that reinforce:
- click impulse
- visual impact
- bingeability
- community activity

## Practical Rules

- Optimize for first-screen conversion.
- Do not overload with too many sections.
- A landing page should feel curated, not exhaustive.
- If only one source asset is strong, repeat it intelligently rather than adding weak visuals.
